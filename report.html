JS

Some text about JS

Did not use any frontend frameworks, implemented our own router and template module. The router performs client-side routing for the SPA. Initialising the router would populate an array of routes with parameters like uri, callback, parameters, regular expression etc. The router then matches the current pathname and parameters to the corresponding route and displays the corresponding content. The router also handles more complex pathnames using .where() with regular expressions that for each parameter. This is implemented for each individual show page. for e.g. http://localhost:8080/shows/1 Modify browser history states so its possible to navigate to previous page using the browser back button. Animation and interaction done with js, for example: svgs outline path drawing, smooth scrolls, page transitions, selection of seats, search bar, carousel navigation, everything. The hiding of navbar when scrolling downwards on the shows and account page, will appear if hovered over and when scrolled upwards. .

Server

Our server uses express, and all the code uses the modern async/await style. It runs both on HTTP (port 8080) and HTTPS (port 8443). The HTTPS certificate is self-signed as the server is not hosted on a domain. Cookies are used for login sessions, with the option of expiring with the browser session or after 7 days if the 'remember me' box is checked. The helmet package was used to set appropriate HTTP headers for security. Cookies are set to HttpOnly, and secure (if on HTTPS).


Database

Our database uses SQLite. A create/drop script (/database/create.sql) can be used to reset the database, and a Node script (/database/init_db.js) can be run to populate it with test values. All databases accesses are done via a module (/database/database.js).
